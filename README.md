# A* 경로 탐색 알고리즘 (C++)

## 프로젝트 소개
이 프로젝트는 C++로 구현한 **A* (A-Star) 경로 탐색 알고리즘**이다.  
2차원 Grid 환경에서 시작 지점(`S`)부터 목표 지점(`G`)까지의 최단 경로를 탐색하며,  
벽(`#`)을 피해 이동 가능한 경로를 계산한다.

## 개발 목적
- A* 알고리즘의 동작 원리 이해
- 2차원 Grid 기반 경로 탐색 구현
- 객체지향적으로 클래스 역할을 분리하여 설계
- STL(`vector`, `priority_queue`) 활용 경험 정리

## 주요 기능
- 2차원 Grid 맵 기반 경로 탐색
- 시작점(S), 도착점(G), 벽(#), 빈 공간(.) 표현
- Manhattan Distance 휴리스틱 적용
- 우선순위 큐를 활용한 탐색
- 탐색 완료 후 최종 경로 복원 및 출력

## 클래스 구조

### GridMap
- 맵 데이터 저장 및 초기화
- 시작점 / 도착점 위치 관리
- 좌표 유효성 검사
- 벽 여부 확인
- 경로 출력

### Node
- 현재 좌표 정보 저장
- g, h, f 비용 저장
- 부모 좌표 정보 저장

### AStarPathfinder
- A* 알고리즘 수행
- 휴리스틱 계산
- 최종 경로 복원

## 사용 기술
- C++
- STL
  - `vector`
  - `priority_queue`
  - `algorithm`
  - `limits`

## 맵 기호 설명

| 기호 | 의미 |
|------|------|
| `S` | 시작 지점 |
| `G` | 도착 지점 |
| `#` | 벽(이동 불가) |
| `.` | 빈 공간 |
| `*` | 최종 탐색 경로 |

## 핵심 알고리즘 설명

A* 알고리즘은 다음 비용을 기준으로 탐색한다.

- `g` : 시작 지점부터 현재 노드까지의 실제 이동 비용  
- `h` : 현재 노드에서 목표 지점까지의 추정 비용  
- `f` : `g + h`  

이 프로젝트에서는 `h` 값을 계산하기 위해 **Manhattan Distance**를 사용하였다.

```cpp
abs(x1 - x2) + abs(y1 - y2)
```

상하좌우 이동만 허용되는 Grid 환경에 적합한 휴리스틱이다.

## 구현 포인트

- `GridMap`, `Node`, `AStarPathfinder`로 역할을 분리하여 설계  
- `priority_queue`를 사용해 가장 작은 `f` 비용을 가진 노드를 우선 탐색  
- `parent` 정보를 저장하여 목표 지점 도달 후 역추적으로 경로 복원  
- 콘솔 출력으로 결과를 직관적으로 확인 가능
